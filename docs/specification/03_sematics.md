# Семантика языка Astra

## 1. Типы данных

Язык Astra — статически типизированный.

| Тип      | Описание                                         | Литералы              | Операции                                     |
|----------|--------------------------------------------------|-----------------------|----------------------------------------------|
| `int`    | целое число со знаком (64 бита)                  | `0`, `42`, `-10`      | `+`, `-`, `*`, `/`, унарный `-`              |
| `float`  | число с плавающей точкой (64 бита, IEEE 754)     | `3.14`, `0.5`, `10.0` | `+`, `-`, `*`, `/`, унарный `-`              |
| `string` | неизменяемая последовательность символов Unicode | `"hello"`, `""`       | конкатенация `+`, индексация `[]`, `len(…)`  |


## 2. Точка входа

Программа состоит из последовательности операторов, разделённых символом `;`. Выполнение начинается с первого оператора верхнего уровня и продолжается последовательно до конца файла.

Точка входа не выделяется специальным синтаксисом; код на верхнем уровне является непосредственно исполняемым.

## 3. Область видимости

Области видимости — лексические (статические). Новая область создаётся **блоком** `start ... end`.

 - Глобальная область — весь файл. Переменные, объявленные на верхнем уровне, видны во всех вложенных блоках (если не перекрыты внутренним объявлением).

- Локальная область — каждый блок start ... end. Переменная, объявленная внутри блока, существует и доступна только от точки объявления до конца этого блока. Вложенные блоки видят все имена из внешних областей (лексическое замыкание без функций в данной версии).
  
- Можно объявить переменную с тем же именем во вложенном блоке — она **перекрывает** внешнюю переменную до конца вложенного блока.

## 4. Ввод / вывод

### Ввод

`input` переменная;

- Ожидает ввода от пользователя из стандартного потока ввода.
- Считанное значение преобразуется к типу переменной и записывается в неё.
- Переменная должна быть объявлена заранее (через let).
- Если введённые данные не соответствуют типу переменной, поведение определяется реализацией (например, приводит к ошибке времени выполнения или запрашивает ввод повторно).

### Вывод

`show` выражение;

- Выводит текстовое представление значения выражения в стандартный вывод.
- После вывода автоматически добавляется перевод строки.
- Значение выражения не сохраняется.

 ## 5. Объявление переменной
## 5.1 Переменная `let`
`let` имя [: тип] [= выражение] ;

`let` имя = выражение ;

- Если указан и тип, и инициализатор, тип выражения должен совпадать с объявленным.
- Если тип опущен, он выводится из типа инициализатора.
- Если тип указан, а инициализатор отсутствует, переменная объявляется без начального значения. **Использование такой переменной до присваивания является семантической ошибкой.**
- Переменная изменяема — ей можно присваивать новые значения.

## 5.2 Константа `const`

`const` имя : тип = выражение ;

- Тип обязателен.
- Инициализатор обязателен, его тип должен совпадать с указанным.
- Константа **неизменяема**. Присваивание константе или её модификация запрещены.
- Константа видна в области видимости блока, где объявлена, и во вложенных блоках.
- Константы находятся в общем пространстве имён с переменными. В одной области видимости нельзя объявить переменную и константу с одинаковым именем.


## 6. Операторы

### 6.1 Присваивание

идентификатор = выражение ;

- Левая часть должна быть именем переменной, объявленной через `let`.
- Тип выражения должен совпадать с типом переменной.
- Присваивание является оператором, а не выражением; цепочечное присваивание (`a = b = 5`) недопустимо.

### 6.2 Выражения

#### 6.2.1 Арифметические операции
- `+`, `-`, `*`, `/` определены для операндов одного числового типа (`int` или `float`).
- Результат имеет тот же тип, что и операнды.
- Деление на ноль — ошибка времени выполнения.
- Унарный `-` допустим для `int` и `float` и возвращает значение того же типа.

#### 6.2.2 Конкатенация строк
- Оператор `+`, применённый к двум операндам типа `string`, выполняет конкатенацию и возвращает новую строку.

#### 6.2.3 Индексация строки
- Выражение `s[i]` допустимо, если `s` имеет тип `string`, а `i` — `int`.
- Индекс должен находиться в диапазоне `0 ≤ i < len(s)`, иначе ошибка времени выполнения.
- Результат — строка из одного символа (тип `string`).
- Присваивание элементу строки запрещено (строки неизменяемы).

### 6.3 Встроенная функция `len`

`len`( выражение )

- Аргумент обязан иметь тип `string`.
- Возвращает целое число (`int`) — количество символов в строке.
- Может использоваться в любом месте, где допустимо выражение.

## 7. Контекстные семантические правила

1. Имя должно быть объявлено до использования в текущей или внешней области.
2. Чтение неинициализированной переменной – ошибка.
3. Тип инициализатора и правой части присваивания должен соответствовать ожидаемому.
4. Конкатенация только `string + string`; смешение типов – ошибка.
5. Индексация – только строки с целым индексом; присваивание элементу – запрещено.
6. Присваивание константе или элементу строки – ошибка компиляции.
7. `input` ожидает существующую переменную.
8. Ключевые слова: `int`, `float`, `string`, `let`, `const`, `show`, `input`, `start`, `end`, `len` запрещены как идентификаторы.
9. Имена встроенных типов (int, float, string) и встроенной функции len зарезервированы; попытка объявить переменную, константу или иной идентификатор с таким именем является семантической ошибкой.
10. Левая часть присваивания (=) может быть только простым идентификатором (именем переменной, объявленной через let). Выражения вида s[i], доступ к членам (.), вызовы функций и т.д. не допускаются, даже если они в конечном счёте ссылаются на изменяемую ячейку.
11. Присваивание константе (или попытка изменить её иным способом) — ошибка компиляции (семантическая), а не только времени выполнения.
    
## 8. Ошибки выполнения
- Деление на ноль.
- Индекс вне границ `[0, len(s)-1]`.
- Несоответствие типа при вводе.

При возникновении ошибки программа немедленно завершается.
