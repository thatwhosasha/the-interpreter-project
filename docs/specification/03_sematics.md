# Семантика языка Astra

Этот документ описывает семантику языка Astra:
* систему типов и их особенности
* смысл всех языковых конструкций
* контекстно-зависимые правила, не выразимые в EBNF
* требования к семантически корректным программам

## 1. Типы данных

Astra является статически типизированным языком с выводом типов для переменных и констант. Каждое выражение имеет тип, известный до выполнения.

### Базовые типы данных

|Тип     | Описание   | Литералы| Операции |
|--------|------------------------------|---------|----------|
|`int `    |Целое число со знаком (64-бит)             |`0`, `42`, `-10`| `+`, `-`, `*`, `/`, сравнения, унарный `-`|    
|`float `  |Число с плавающей точкой (64-бит, IEEE 754)|`3.14`, `0.5`, `10.0`| `+`, `-`, `*`, `/`, сравнения, унарный `-`|
|`string ` |Неизменяемая последовательность символов Unicode|`"hello"`, `"world"`| конкатенация `+`, индексация `[]`, функция `len()`|
|`bool`    |Логическое значение           |`true`, `false` | `&&`, `\|\|`, `!` |    
|`void`    |Отсутствие значения; используется только как возвращаемый тип функций| | |      

### Литералы

* Литерал целого числа: `42`, `-10`, `0`
* Литерал числа с плавающей точкой: `3.14`, `0.5`, `10.0`
* Литерал строки: `"Hello"`, `"Astra"`
* Логический литерал: `true`, `false`

### Логические значения

Истинность и ложность

В условиях допускаются только значения типа `bool`:
* `true` интерпретируется как «истина»
* `false` интерпретируется как «ложь»

Логические операции

* Логические операции `&&` (И), `||` (ИЛИ) возвращают значение типа `bool`
* Логическая операция `!` (НЕ) возвращает значение типа `bool`
* Логические операции вычисляются по короткой схеме:
  * `&&` — если левый операнд `false`, то правый операнд не вычисляется
  * `||` — если левый операнд `true`, то правый операнд не вычисляется

## 2. Операторы

### Арифметические операторы
Для арифметических операторов `+`, `-`, `*`, `/`:
* Операнды должны быть одного числового типа (int или float)
* Результат имеет тот же тип, что и операнды
* Деление на ноль приводит к ошибке выполнения
  
### Операторы сравнения
Для операторов сравнения `<`, `>`, `<=`, `>=`:
* Операнды должны быть оба целыми числами, оба числами с плавающей точкой или обе строками
* Для строк сравнение выполняется лексикографически
* Результат — значение типа `bool`

Для операторов равенства `==` и `!=`:
* Допускаются любые операнды одинакового типа
* Объекты сравниваются по ссылке
* Результат — значение типа `bool`

### Конкатенация строк
Оператор `+` перегружен для строк:
* Если оба операнда имеют тип string, выполняется конкатенация
* Результат — значение типа string

### Присваивание
* Присваивание возвращает присвоенное значение (правоассоциативно)
* Конструкции вида `a = b = 5` допустимы и означают `a = (b = 5)`
* Выражение слева от присваивания должно быть `lvalue`
* `lvalue` — это имя переменной, объявленной через `let`

### Индексация строк
* Выражение слева от `[]` должно иметь тип `string`
* Индекс (выражение внутри `[]`) должен иметь тип `int`
* Индекс должен находиться в диапазоне от `0` до `длина_строки - 1`
* Результат — строка из одного символа (тип `string`)
* Присваивание по индексу запрещено (строки неизменяемы)

## 3. Области видимости

В языке Astra используются лексические, а не динамические области видимости.

Есть два способа создать новую область видимости:
* Объявить функцию — все формальные параметры и локальные переменные доступны только в области видимости функции
* Объявить блок с помощью `start...end` — все объявления внутри блока видны только в нём
* 
### Захват переменных
Функция захватывает все переменные внешней области видимости и имеет к ним доступ при вызове.

###  Время жизни переменных
* Локальные переменные существуют только в течение выполнения своей области видимости
* После выхода из блока или функции локальные переменные становятся недоступны
* Глобальные переменные существуют в течение всей программы

Пример:

```Astra
let x = 10;  // глобальная x

func foo() -> void
start
    let x = 20;  // локальная x перекрывает глобальную
    show x;      // выводит 20
end
```

## 4. Уникальность имен

Имена переменных, констант и функций находятся в одном пространстве имён.

Действуют следующие правила:
* В одной области видимости все имена должны быть уникальны:
  * нельзя использовать имя для переменной или функции, если оно уже занято другой переменной или функцией
* Во вложенной области видимости можно объявлять символ с именем, совпадающим с именем из внешней области видимости — в этом случае новый символ перекрывает старый 
* мена встроенных типов (`int`, `float`, `string`, `bool`) зарезервированы и перекрывать их нельзя

## 5. Параметры функций

###  Передача параметров
* Параметры типов `int`, `float`, `bool` передаются по значению
* Параметры типа `string` передаются по значению

### Использование параметров
* Параметры можно использовать в левой части присваиваний, как и обычные переменные
* Параметр можно скрыть локальной переменной с тем же именем
  
### Вычисление аргументов
Фактические параметры вычисляются слева направо.

### Соответствие параметров
При вызове функции количество и типы фактических параметров должны точно совпадать с количеством и типами формальных параметров.

## Встроенная функция `len()`
* Синтаксис: `len(выражение)`
* Аргумент должен иметь тип `string`
* Возвращает длину строки как значение типа `int`

## Условный оператор `if...else`
* Выражение условия ветвления должно возвращать тип `bool`
* Если условие истинно, то выполняется блок после `if`
* Если условие ложно и есть ветка `else`, то выполняется блок после `else`
* Ветка `else` необязательна
* Допускается каскадирование условий с помощью вложенных `if` в ветке `else`

## 6. Циклы

### Цикл `while`
* Выражение условия цикла должно возвращать тип `bool`
* Пока условие истинно, тело цикла выполняется
* Перед каждой итерацией вычисляется условиеS
* Тело цикла может быть пустым

### Операторы `break` и `continue`
* `break` — немедленно завершает выполнение ближайшего цикла `while`
* `continue` — завершает текущую итерацию цикла и переходит к проверке условия для следующей итерации
* Использование `break` или `continue` вне цикла запрещено (семантическая ошибка)

## Рекурсия

* Поддерживается рекурсия — функция может вызывать сама себя.
* Поддерживается взаимная рекурсия — функции могут вызывать друг друга, если они объявлены в одной области видимости и между их объявлениями нет объявлений иных категорий символов (например, переменных).

## Оператор `show`
* Вычисляет выражение и выводит его текстовое представление в стандартный вывод
* Результат отбрасывается

## Оператор `return`

* `return` завершает выполнение текущей функции и передаёт управление вызывающему коду.
* Если функция объявлена с типом возврата, отличным от `void`, то `expression` обязательно и его тип должен соответствовать объявленному типу возврата. Если `expression` отсутствует, это семантическая ошибка.
* Если функция объявлена с типом возврата `void`, то `expression` не допускается (т.е. используется только `return;`). Отсутствие `return` в `void`-функции не является ошибкой — функция завершается после выполнения последней инструкции.
* Выполнение `return` в функции с типом возврата, отличным от `void`, без `expression` — семантическая ошибка.
* Выполнение `return` с `expression` в функции с типом возврата `void` — семантическая ошибка.
